# CellSentry Test Cases

Community-contributed test fixtures for improving CellSentry's detection accuracy.

## Purpose

These `.xlsx` files contain synthetic spreadsheet patterns that CellSentry should detect. They serve two purposes:

1. **Regression testing** — ensure known patterns stay detected across versions
2. **Training data** — edge cases help improve the local AI model in future releases

## Privacy Rules

- **Use synthetic data only** — never include real names, phone numbers, emails, or financial data
- All data must be clearly fictional (e.g., "Jane Doe", "555-0100", "john@example.com")
- If in doubt, describe the *pattern* in an [issue](https://github.com/almax000/cellsentry/issues/new?template=detection-miss.yml) instead of submitting a file

## Directory Structure

```
test-cases/
├── audit/          # Formula audit edge cases
├── pii/            # PII detection edge cases
└── extraction/     # Document extraction edge cases
```

## Contributing a Test Case

1. Create an `.xlsx` file using Excel, LibreOffice, or a script
2. Add a comment in cell **A1** explaining what the file tests
3. Place it in the appropriate subdirectory
4. Submit a PR (see [CONTRIBUTING.md](../publish/CONTRIBUTING.md))

### Format

- One test scenario per file
- Descriptive filename: `vlookup-if-nesting.xlsx`, not `test1.xlsx`
- Keep files small — a few rows is enough to demonstrate the pattern

## Regenerating Fixtures

The example fixtures are generated by a script:

```bash
cd app && npx tsx ../scripts/generate-test-cases.ts
```
